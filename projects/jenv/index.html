What’s jEnv ?

jEnv is a command line tool to help you forget how to set the JAVA_HOME environment variable.

Quick start

1. Installation

git clone https://github.com/gcuisinier/jenv.git ~/.jenv

or on a Mac :

brew install jenv

2. Enabling jEnv in your shell

Bash

$ echo 'export PATH="$HOME/.jenv/bin:$PATH"' >> ~/.bash_profile
$ echo 'eval "$(jenv init -)"' >> ~/.bash_profile

Zsh

$ echo 'export PATH="$HOME/.jenv/bin:$PATH"' >> ~/.zshrc
$ echo 'eval "$(jenv init -)"' >> ~/.zshrc

3. Configuration of JDKs Installation

$ jenv add /System/Library/Java/JavaVirtualMachines/1.6.0.jdk/Contents/Home
  oracle64-1.6.0.39 added
$ jenv add /Library/Java/JavaVirtualMachines/jdk17011.jdk/Contents/Home
  oracle64-1.7.0.11 added

4. Usage

You can configure a global (default one) JDKs :

jenv global 1.7

But also by directory (and so, by project) :

jenv local 1.6

This will add a .java-version file in this directory

And if you want to have the JAVA_HOME environment variable set automatically, you just have to enable the export plugin :

jenv enable-plugin export

jEnv can also manage the JAVA_OPTS, MAVEN_OPTS !
